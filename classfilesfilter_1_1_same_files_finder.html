При создании SameFilesFinder выполняется алгоритм поиска дубликатов, который состоит из двух этапов:

  1. Формирование списка файлов для сравнения (createFileList()). Алгоритм createFileList():
    1) Преобразование списков included, excluded в формат boost::filesystem::path (path_t) 2) Проход по всем included. В зависимости от level - рекурсивный проход (с вхождениями во вложенные директории), либо нерекурсивный (про текущей директории). 3) Фильтр каждого вхождения по критериям:
    • вхождение является файлом;
    • размер файла не менее minsize;
    • имя файла сочетсатся с маской mask. В итоге формируется список файлов, прошедших все фильтры, который можно получить при помощи метода fileList() или вывести в поток при помощи printFileList().
  2. Формирование групп файлов-дубликатов (findSameFiles()) Алгоритм (findSameFiles()):
    1) Если количество файлов <= 1, досточный выход из функции, так как снавнивать не с чем. Выполняется требование ТЗ: блок файла читается только в случае необходимости. 2) Формируется ассоциативный массив, где ключом является хэш прочитанного из файла блока, а значением - список файлов, для которых данное хэш-значение одинаково. 3) Формируется список файлов, чтение которых завершено. Если таких файлов оказалось более 1, то это значит, что все предыдущие блоки для них одинаковы, и их можно объединить в группу и записать в итоговый результат. 4) Для всех списков файлов из ассоциативного массива, размером больше 1 вызываем функцию рекурсивно. В итоге формируется список групп файлов-дубликатов, который можно получить при помощи метода sameFilesGroups() или вывести в поток при помощи printSameFiles().
    Parameters
    included- список директорий для сканирования
    excluded- список директорий для исключения из сканирования
    mask- маска сканирования
    level- уровень сканирования
    minsize- минимальный размер файлов
    blocksize- размер блока, по которому производится четние каждого файла
    hashalg- алгоритм хэширования читаемого из файла блока: 0 - MD5, 1 - CRC32
